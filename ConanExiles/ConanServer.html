Admin Commands

Action Controls Description
Open Admin Panel [CTRL]+[SHIFT]+[C]
-or-
[SHIFT]+[INSERT]		 Opens the Admin Panel GUI
Open Console Commands Interface [INSERT]
-or-
[~]		 Opens the Console Commands Interface, where you can type Console Commands
Open Full Console Commands Interface Double press [INSERT]
-or-
Double press [~]		 Open the Full Console Commands Interface, show history of parsed commands
Get Precise Coördinates [R-CTRL]+[R-SHIFT]+[R-ALT]+[L] Opens GUI that readout your Coördinates in the Gameworld
Kill / Destroy Target [SHIFT]+[DELETE] Kills or Destroys a target under your Cursor:
  • Removes a single Building Part
  • Kills a single NPC
  • Does NOT delete World-Chunks, nor Campfires, Forges etc.
Kill / Destroy Multiple Targets [CTRL]+[SHIFT]+[DELETE] Kills or Destrots Multiple targets under your Cursor, with a larger radius:
  • Removes multiple Building Part
  • Kills multiple NPC, also Pets
  • Does delete World-Chunks
Infinite Building Item Stacks [CTRL]+[ALT]+[SHIFT]+[F10] Infinite Building Item Stacks in your Inventory, you must have at least one quantity per related item in your inventory
Start Sandstorm [CTRL]+[ALT]+[SHIFT]+[X] Starts a Sandstorm, it will take a few minutes to reach your position
Command Description
"Admin Manipulation"
MakeMeAdmin Enters Admin mode. Triggers Admin Rights Granted
MakeMeNormal Exits Admin mode
God Toggle God mode. Invulnerable to all damage, knockbacks, status effects etc. Only exception is the Cursewall
Cloak Toggle Cloak Mode. Invisible to NPCs, even when attacking them
Invisibility Toggle Invisibility Mode. Makes your character invisible
NoSprintCost Toggle No Sprinting Stamina Cost. No stamina drain while sprinting. Not unlimited stamina
Fly Grants Fly mode. Fly mode has two different poses, "Fly" and "FlyB", which are occasionally switched
Ghost Grants Ghost mode.
  • Grants Fly mode (See Fly)
  • Grants Noclip, you can pass through objects
  • Building is possible, but building pieces will not snap to other building pieces
Walk Exits the Fly or Ghost mode
"Server Manipulation"
ToggleDebugHUD Open Player Data GUI. Admin mode not required.
  • Displays current location, in the lower left of the screen
  • Displays other data, including ping, in the upper right of the screen
GetServerHealth Lists server statistics in the console. Toggle the Full Console Commands Interface to see the Output:
  • FPS; Frames Per Second of the Server
  • Number of Players Connected to the Server
  • Number of Spawned Actors in the Server
  • Number of Player Bases, including detached foundations
  • Number of Building-Parts in the Server. Parts include: Foundations, Walls, Roofs. Possibly also includes: Chests, CraftingStations etc.
  • LOD Count; Level of Detail
GetAllServerSettings Displays a list, of all available server settings you can modify, in the console. Toggle the Full Console Commands Interface to see the Output
SetServerSetting "[ServerSettingName]" [Value] Quotations are required.
Allows you to modify the Server Settings inside ServerSettings.ini while in-game. Includes server settings not changeable in Server Settings tab
(Optional) For admins using Rcon, Server Settings can be set remotely via this command as well

INI Settings Legend

How To Read

Headers:

  • * = Recommended Settings
  • 1 = Civilized Preset Setting
  • 2 = Decadent Preset Setting
  • 3 = Barbaric Preset Settings
    •

    Colors:

  • White = INI Changeable Only
  • Gray = In-Game Changeable
  • Purple = Unknown Settings
  • Red = Broken Settings

    Server Messaging System

    You can use the three warming messages to automatically send a message to all the connected players:

      Rcon needs to be enabled (with a valid password, port, etc…)
      The time is the number of hours or minutes relative to the expected restart time

    Note that since the delay accepts up to a 24 hours range, you can creatively use the feature to inform your players of regular events, advertise things, etc… the only limit is your imagination and the amount of bugs you may find when playing with it.

    Messages can also contain “macros”, which are symbols that get expanded when displayed.
    As of now there are only two available macros, but they are practical to send useful information to the players:

      {servername} - expanded to what you have set up in the Server Name field
      {steamconnect} - expanded to a connection string players can click on to connect
      {steamconnect_nopassword} - same thing, but without the password being displayed
      {reason} - expanded to display the reason for the server restart (mod update, automated, etc…)
      {ip} - replaced by the external IP address of the server
      {game_port} - replaced by the value of the game port
      {steam_port} - replaced by the value of the steam query port
      {rcon_port} - replaced by the value of the rcon port
      {server_password} - replaced by the server password
      {admin_password} - replaced by the admin password (beware!)
      {rcon_password} - replaced by the rcon password (beware!)

    Example setting the “Ready” message to:

      The {servername}'s server is now open, please join on {steamconnect}

    will allow your Discord player to see the following clickable message:

      The Toolguy’s server is now open, please join on steam://connect/123.231.132.213:27015/pass

    Remember: The message sending feature requires Rcon and/or Discord WebHook to be enabled!
